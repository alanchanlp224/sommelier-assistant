Sommelier Assistant — Chrome "Load unpacked"

WHY "missing manifest.json" IF YOU UNZIPPED FROM GITHUB
  GitHub gives you SOURCE CODE only. Chrome needs the built extension in the
  "dist" folder. Build first (see INSTALL.txt in the repository root), then
  load the "dist" folder — not the repository root.

Use THIS folder as the unpacked extension root
  The folder must contain manifest.json, background.js, and content.js together.
  That is the "dist" output after: npm install && npm run build

If you received SOURCE only (cloned or GitHub ZIP, no usable dist yet):
  1. Install Node.js LTS from https://nodejs.org/
  2. Open a terminal in the project folder (Windows: Command Prompt or PowerShell).
  3. Run: npm install
  4. Run: npm run build
  5. In chrome://extensions enable Developer mode → Load unpacked → choose the
     **dist** subfolder inside this project (not the project root).

If you downloaded sommelier-assistant.zip from a GitHub Release:
  Unzip it and Load unpacked that folder — it is already built.

To create the release zip yourself (any OS): npm run package
